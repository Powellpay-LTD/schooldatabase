School Management System - Database Design Requirements
Project Overview
Building a School Management System (SSMS) for Ugandan schools following the new Competency-Based Curriculum (CBC).

2. CBC Context (Uganda)
Rolled out starting S1 in 2020

First UCE cohort under CBC sat in 2024

Subjects have themes/topics with specific learning outcomes (competencies)

Continuous Assessment (CA) AND summative exams

No learner "fails" outright - certificate requires at least a D in one subject

Generic Skills rated every term (Beginning → Developing → Proficient → Mastery)

3. Grading System
A = Exceptional

B = Outstanding

C = Satisfactory

D = Basic

E = Elementary

Final Score = CA score + End of Term score

Design Considerations
18 tables total identified

CBC curriculum requires tracking of competencies, not just grades

Need to handle continuous assessment separately from end of term exams

Generic skills must be rated each term

Students belong to one stream per academic year

Students can have multiple guardians (many-to-many relationship)

Staff members have specific roles and permissions

Expected Outputs Per Issue
CREATE TABLE statements with naming conventions

Realistic sample data (Uganda context - real subjects, names, terms)

Working queries that answer specific questions

Design write-up explaining choices and alternatives considered

Research citations for CBC/UNEB content

Workflow Requirements
One branch per issue

PRs reviewed by teammate before merging

Documentation of design trade-offs
